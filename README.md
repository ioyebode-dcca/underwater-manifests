# underwater-manifests

GitOps repository watched by Flux for the Octopus Underwater App.

## How This Works

Jenkins pushes code → builds image → updates image tag in this repo → Flux detects change → deploys to EKS.

You never run `kubectl apply` manually. Git is the source of truth.

## Repository Structure

```
underwater-manifests/
├── clusters/
│   ├── dev/              # Flux watches this for dev cluster
│   │   ├── flux-system/  # Auto-generated by flux bootstrap
│   │   └── kustomization.yaml
│   └── prod/             # Flux watches this for prod cluster
│       ├── flux-system/
│       └── kustomization.yaml
├── apps/
│   ├── base/
│   │   └── underwater/   # Base deployment, service, image policy
│   └── overlays/
│       ├── dev/          # Dev overrides: 1 replica, SPOT, dev-* image tags only
│       └── prod/         # Prod overrides: 3 replicas, ON_DEMAND, release-* tags only
└── infrastructure/
    ├── ingress-nginx/
    └── monitoring/
```

## Image Tag Separation

Dev cluster only ever pulls `dev-*` tagged images.
Prod cluster only ever pulls `release-*` tagged images.
This is enforced by the ImagePolicy in each overlay.

## Bootstrap Commands

### Dev cluster
```bash
flux bootstrap github \
  --owner=YOUR_GITHUB_USERNAME \
  --repository=underwater-manifests \
  --branch=main \
  --path=clusters/dev \
  --personal
```

### Prod cluster
```bash
flux bootstrap github \
  --owner=YOUR_GITHUB_USERNAME \
  --repository=underwater-manifests \
  --branch=main \
  --path=clusters/prod \
  --personal
```

## Verify Flux is Working
```bash
flux get all
kubectl get pods -n flux-system
flux logs --follow
```

## Manual Sync (force Flux to reconcile now)
```bash
flux reconcile kustomization flux-system --with-source
```
